# Contributing

## Development Flow

1. Use Node.js 18+.
2. Run `npm install`.
3. Start local dev with `npm run dev`.
4. Run `npm run check` before opening a PR.
5. Run `npm run test` before opening a PR.

## Commit Style

Use conventional prefixes:

- `feat:`
- `fix:`
- `docs:`
- `refactor:`
- `test:`
- `chore:`

## Pull Request Checklist

- Keep changes focused and typed.
- Maintain strict TypeScript compatibility.
- Ensure tests pass (`npm run test`).
- Update README for user-facing behavior changes.
- Include screenshots or API examples for UI/API changes.
